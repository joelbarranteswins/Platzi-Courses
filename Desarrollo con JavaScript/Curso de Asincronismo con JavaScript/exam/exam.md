
## Examen 📌


1. ¿Para qué nos sirve el método "catch()"?

	- 📌 Registra la razón del rechazo.


2. ¿Cómo aseguramos manejar los errores asincrónicos correctamente?

	- 📌 `try { ...código } catch (error) { ...código }`


3. Una función callback es:

	- 📌 Una función que se pasa a otra función como un argumento, invocada dentro de la función externa.


4. ¿Nos permite ejecutar una serie de promesas secuencialmente?

	- 📌 `Promise.all()`


5. ¿Cuál es el método recomendado por la comunidad para manejar asincronismo en JavaScript?

	- 📌 Async/await

6. Palabra que nos permite definir una función así­ncrona. Selecciona la opción correcta:

	- 📌 async

7. ¿Cuáles son los argumentos que recibe una promesa?

	- 📌 resolve, reject


8. ¿Cuál es la expresión que pausa la ejecución de la función así­ncrona y espera la resolución de la promise?

	- 📌 await


9. ¿El método then() retorna?

	- 📌 Promesa

10. ¿Para qué nos sirve xhttp.status === 200?

	- 📌 Verificamos que el estatus de la petición HTTP resuelva el estado 200.


11. Es el problema de anidamiento de callbacks, que las promesas resuelven. Selecciona la respuesta correcta.

	- 📌 callback hell


12. ¿Cuál es la forma correcta de retornar un Error en reject?

	- 📌 `reject (new Error(`Error`))`


13. ¿Para qué utilizamos JSON.parse(xhttp.responseText)?

	- 📌 Convertir una respuesta de texto en un Objecto iterable.

14. ¿Para qué nos sirve la clase XMLHttpRequest?

	- 📌 Nos permite realizar solicitudes HTTP de una forma muy fácil.
