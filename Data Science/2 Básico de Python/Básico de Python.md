## Interrumpiendo ciclos con break y continue

**Active Recall**

- ¬øC√≥mo podemos modificar el funcionamiento normal de un ciclo?
    
    Utilizando:
    
    - `break`: al momento de llegar a un valor corta el ciclo
    - `continue`: nos permite saltarnos la vuelta del ciclo si se cumple una condici√≥n, lo que este debajo de esta palabra no se ejecutar√°.
- ¬øQu√© significa modificar el funcionamiento normal de un ciclo?
    - Ejecutar vueltas del ciclo y otras no
    - Llegar solo a una determinada vuelta del ciclo
    - Terminal el ciclo sin haber llegado al final.
- ¬øC√≥mo debemos de implementar `break` y `continue`?
    
    En ciclos `for`o `while` con el uso de condicionales.
    

**Notes:**

Hasta ahora hemos aprendido c√≥mo funcionan los ciclos, sin embargo, debemos conocer que dentro de Python y dentro de cualquier lenguaje de programaci√≥n se puede modificar un funcionamiento normal del ciclo, es decir, podemos ejecutar ciertas vueltas del ciclo y otras no o llegar a determinada vuelta y terminar el ciclo sin necesidad de terminar el ciclo. Esto lo podemos hacer con `break` y `continue`.

Para aprender mejor c√≥mo funcionan estos operadores veremos tres ejemplos:

- Imprimir solo n√∫meros pares utilizando `continue`. Lo que nos dice es que lo que esta despu√©s de continue no se va a ejecutar y pasa al siguiente n√∫mero.

```python
def run():

        if contador % 2 != 0: # Esto se lee c√≥mo si al contador al dividirlo por 2, el resto de la divis√≥n es distinto de 0 nos saltamos esa vuelta del ciclo
            continue # lo que est√° debajo de esta continue no se va a ejecutar
        print(contador)

if __name__ == '__main__':
    run()
```

- Imprimir n√∫meros solo hasta llegar a uno en espec√≠fico y despu√©s de ello cortarlo con `break`.

```python
def run():
		for i in range(10000):
        print(i)
        if i == 5678:
            break # Lo que hace break es al momento de llegar a este n√∫mero se corta el ciclo

if __name__ == '__main__':
    run()
```

> La variable `i` es un est√°ndar dentro de la programaci√≥n para usarlo en variables que va cambiando de valor.
> 
- Listar las letras de una palabra y cortar el ciclo al encontrar una letra en espec√≠fico con `break`.

```python
def run():
		texto = input('Escribe un texto: ')
    for letra in texto: 
        if letra == 'o':
            break
        print(letra)

if __name__ == '__main__':
    run()
```

üìå **SUMMARY:** dentro de los ciclos podemos utilizar dos palabras muy poderosas para modificar el funcionamiento normal del ciclo, estas son: `break` para al llegar a un valor cortar el ciclo y `continue` para saltarnos la vuelta del ciclo si se cumple una condici√≥n. Estos ciclos los podemos utilizar en todas las funcionalidades que tenemos de nuestro ciclo while y for.

Lo importante es que hagamos las divisiones de forma entera, es decir, teniendo cociente y resto de la divisi√≥n, nunca vamos a tener un resto que vaya a ser igual a cero, solo cuando dividimos por 1 y por su mismo n√∫mero.

```python
def es_primo(numero):  # Creamos una funci√≥n para identificar si un n√∫mero es primo
		contador = 0 

		for i in range(1, numero + 1):  # Colocamos el + 1 porque el √∫ltimo par√°metro de la funci√≥n range no se toma en cuenta dentro del rango
				if i == 1 or i == numero:
						continue    # Lo que quiere decir est√° linea es que si i es igual a 1 o i es igual al n√∫mero nos saltamos la l√≠nea
				if numero % i == 0:
						contador += 1
		if contador == 0:  # Lo que hamos es preguntar si el contador es igual a cero, es decir, si al dividir el n√∫mero por todos los n√∫meros dio todas las divisiones inexactas.
				return True 
		else: 
				return False

def run():
		numero = int(input('Escribe un n√∫mero: '))
		if es_primo(numero): # Esta es una funci√≥n que vamos a crear para determinar si el n√∫mero es o no un n√∫mero primo

		else: 
				print('No es primo')

if __name__ == '__main__': 
		run()
```

> Cu√°ndo tenemos dentro de un condicional if lo siguiente: `if es_primo(numero) == True:`. Podemos obviar poner el `== True`, ya que dentro de Python es opcional ponerlo. Python entiende que le estamos preguntando si el resultado de la funci√≥n es verdadera.

```python
import random

def generar_contrasena():
    mayusculas = ['A', 'B', 'C', 'D', 'E', 'F', 'G', 'H', 'I', 'J', 'K', 'L', 'M', 'N', '√ë', 'O', 'P', 'Q', 'R', 'S', 'T', 'U', 'V', 'X', 'Y', 'Z']

    simbolos = ['*', '+', '-', '/', '@', '_', '?', '!', '[', '{', '(', ')', '}', ']', ',', ';', '.', '>', '<', '~', '¬∞', '^', '&', '$', '#', '"']

		caracteres = mayusculas + minusculas + simbolos + numeros # Sumaremos a todas nuestras listas en una lista √∫nica
		contrasena = [] # Generamos una lista vacia en donde vamos a poner caracteres aleatorios para generar nuestra contrase√±a.

    for i in range(15):

				contrasena.append(caracter_random) # Lo que hace append es agregar al caracter random en contrasena

		contrasena = "".join(contrasena) # Esto lo que nos permite es generar un string de nuestra lista original
		return contrasena # Esto nos permite que al ejecutarse la funci√≥n run la contrase√±a se guarde en contrasena

def run():
    contrasena = generar_contrasena()
    print('Tu nueva contrasena es: ' + contrasena)

if __name__ == '__main__':
    run()
```
